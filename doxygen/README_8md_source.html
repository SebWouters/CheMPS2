24 Information
25 -----------
26 
27 CheMPS2 is a scientific library which contains a spin-adapted implementation
28 of the density matrix renormalization group (DMRG) for ab initio quantum
29 chemistry. This wavefunction method allows to obtain numerical accuracy in
30 active spaces beyond the capabilities of full configuration interaction (FCI),
31 and allows to extract the 2-, 3-, and 4-particle reduced density matrices
32 (2-, 3- and 4-RDM) of the active space.
33 
34 For general active spaces up to 40 electrons in 40 orbitals can be handled
35 with DMRG, and for one-dimensional active spaces up to 100 electrons in 100
36 orbitals. The 2-RDM of these active spaces can also be easily extracted,
37 while the 3- and 4-RDM are limited to about 28 orbitals.
38 
39 When the active space size becomes prohibitively expensive for FCI, DMRG can
40 be used to replace the FCI solver in the complete active space self consistent
41 field (CASSCF) method and the corresponding complete active space second order
42 perturbation theory (CASPT2). The corresponding methods are called DMRG-SCF
43 and DMRG-CASPT2, respectively. For DMRG-SCF the active space 2-RDM is required,
44 and for DMRG-CASPT2 the active space 4-RDM.
45 
46 CheMPS2 is designed for high-performance computers, with a hybrid
47 parallelization for mixed distributed and shared memory architectures,
48 realized with the Message Passing Interface ([MPI](http://www.mpi-forum.org/))
49 and the Open Multi-Processing ([OpenMP](http://openmp.org/wp/)) API.
50 
51 The CheMPS2 library can be interfaced with quantum chemistry codes which
52 can handle R(O)HF calculations and molecular orbital matrix elements. This
53 has been done for [psi4](http://www.psicode.org) and
54 [pyscf](https://github.com/sunqm/pyscf), as described in the
55 [user manual](http://sebwouters.github.io/CheMPS2/index.html). Usage of the
56 library is illustrated in the [c++](#4-test-libchemps2) and
57 [python](#7-test-pychemps2) tests.
58 
59 The CheMPS2 binary allows to perform DMRG-SCF and DMRG-CASPT2 calculations
60 based on input files. Molecular orbital matrix elements should then be
61 provided in FCIDUMP format. Usage of the binary is illustrated in the
62 [binary](#5-test-the-chemps2-binary) example.

153 Installation
154 ------------
155 
156 ### 1. Dependencies
157 
158 CheMPS2 can be built with [CMake](http://www.cmake.org/) and depends on
159 
160 - BLAS
161 - LAPACK
162 - GSL ([GNU Scientific library](http://www.gnu.org/software/gsl/))
163 - HDF5 ([Hierarchical Data Format Release 5](http://www.hdfgroup.org/HDF5/))
164 
165 It is parallelized for shared memory architectures with the Open
166 Multi-Processing ([OpenMP](http://openmp.org/wp/)) API and for
167 distributed memory architectures with the Message Passing Interface
168 ([MPI](http://www.mpi-forum.org/)). A hybrid combination of both
169 parallelization strategies is supported.
170 
171 ### 2. Download
172 
173 It is advised to clone the CheMPS2 git repository from github. In your
174 terminal, run:
175 
176  > cd /sourcefolder
177  > git clone 'https://github.com/sebwouters/chemps2'
178  > cd chemps2
179 
180 That way, future updates and bug fixes can be easily pulled in:
181 
182  > cd /sourcefolder/chemps2
183  > git pull
184 
185 ### 3. Build the chemps2 library and binary
186 
187 The files
188 
189  /sourcefolder/chemps2/CMakeLists.txt
190  /sourcefolder/chemps2/CheMPS2/CMakeLists.txt
191  /sourcefolder/chemps2/tests/CMakeLists.txt
192  /sourcefolder/chemps2/sphinx/CMakeLists.txt
193 
194 provide a minimal compilation. In your terminal, run:
195 
196  > cd /sourcefolder/chemps2
197  > mkdir build
198  > cd build
199 
200 CMake generates makefiles based on the user’s specifications:
201 
202  > CXX=option1 cmake .. -DMKL=option2 -DCMAKE_INSTALL_PREFIX=/option3 -DWITH_MPI=option4
203 
204 1. Option1 is the `c++` compiler; typically `g++`, `icpc`, or `clang++`
205  on Linux. It is advised to use the intel compiler if available.
206 2. Option2 can be `ON` or `OFF` and is used to switch on the intel math
207  kernel library.
208 3. /option3 is the prefix of the installation directory; typically `/usr`
209  or `/usr/local` on Linux. On my computer, libchemps2 is then installed
210  in `/option3/lib/x86_64-linux-gnu`, the headers in
211  `/option3/include/chemps2`, and the binary in
212  `/option3/bin/chemps2`.
213 4. Option4 can be `ON` or `OFF` and is used to switch on the possibility
214  to compile with MPI. Please note that the compiler should then provide
215  `mpi.h`. Option1 should hence be the `mpic++` compiler; typically
216  `mpic++` or `mpiicpc` on Linux. It is advised to use the intel compiler
217  if available.
218 
219 If one or more of the required libraries are not found, use the command
220 
221  > CMAKE_INCLUDE_PATH=option5 CMAKE_LIBRARY_PATH=option6 CXX=option1 cmake .. -DMKL=option2 -DCMAKE_INSTALL_PREFIX=/option3 -DWITH_MPI=option4
222 
223 instead, where option5 and option6 are respectively the missing
224 colon-separated include and library paths:
225 
226  CMAKE_INCLUDE_PATH=/my_libs/lib1/include:/my_libs/lib2/include
227  CMAKE_LIBRARY_PATH=/my_libs/lib1/lib:/my_libs/lib2/lib
228 
229 For operating systems based on debian, the HDF5 headers are located in
230 the folder `/usr/include/hdf5/serial`. If CMake complains about the HDF5
231 headers, try to pass it with the option
232 `-DHDF5_INCLUDE_DIRS=/usr/include/hdf5/serial`.
233 
234 To compile, run:
235 
236  > make
237 
238 To install, run:
239 
240  > make install
241 
242 For non-standard installation directories, please remember to append the
243 library path to `LD_LIBRARY_PATH` in your `.bashrc`.
244 
245 ### 4. Test libchemps2
246 
247 To test libchemps2 for compilation **without MPI**, run:
248 
249  > cd /sourcefolder/chemps2/build
250  > make test
251 
252 To test libchemps2 for compilation **with MPI**, run:
253 
254  > cd /sourcefolder/chemps2/build/tests
255  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./test1
256  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./test2
257  ...
258 
259 `YYY` specifies the number of threads per process and `ZZZ` the number of
260 processes. Note that the tests are too small to see (near) linear scaling
261 with the number of cores, although improvement should still be noticeable.
262 
263 ### 5. Test the chemps2 binary
264 
265 To test the chemps2 binary for compilation **without MPI**, run:
266 
267  > man /sourcefolder/chemps2/chemps2.1
268  > cd /sourcefolder/chemps2/build/CheMPS2
269  > ./chemps2 --help
270  > cp /sourcefolder/chemps2/tests/test14.input .
271  > sed -i "s/path\/to/sourcefolder\/chemps2\/tests\/matrixelements/" test14.input
272  > cat test14.input
273  > ./chemps2 --file=test14.input
274 
275 Note that when you use the CASPT2 checkpoint, and want to restart a
276 calculation at a later point, you should
277 
278  1. switch the option `SCF_ACTIVE_SPACE` to `I`
279  2. remove the `CheMPS2_DIIS.h5` checkpoint
280 
281 in order to ensure that **exactly** the same orbitals are used in the
282 different runs.
283 
284 To test the chemps2 binary for compilation **with MPI**, prepend the binary with:
285 
286  > OMP_NUM_THREADS=YYY mpirun -np ZZZ ./chemps2 --file=test14.input
287 
288 ### 6. Build PyCheMPS2
289 
290 PyCheMPS2 is a python interface to libchemps2, for
291 compilation **without MPI**. It can be built with
292 [Cython](http://cython.org/). The installation is independent of
293 CMake and assumes that you have installed the CheMPS2 library with
294 `make install`. For non-standard installation directories of CheMPS2,
295 please remember to append the library path to `LD_LIBRARY_PATH` in
296 your `.bashrc`. In addition, the include path should be appended
297 to `CPATH`:
298 
299  > export CPATH=${CPATH}:/option3/include
300 
301 where `/option3` is the option provided to CMake with
302 `-DCMAKE_INSTALL_PREFIX=/option3` above. For operating systems based
303 on debian, the HDF5 headers are located in the folder
304 `/usr/include/hdf5/serial`. If it was explicitly passed to CMake, it
305 should also be appended to `CPATH`:
306 
307  > export CPATH=${CPATH}:/option3/include:/usr/include/hdf5/serial
308 
309 The python wrapper can be installed with:
310 
311  > cd /sourcefolder/chemps2/PyCheMPS2
312  > python setup.py build_ext -L ${LD_LIBRARY_PATH}
313  > python setup.py install --prefix=/option3
314 
315 On my machine, the python wrapper is installed to the folder
316 `/option3/lib/python2.7/site-packages`, but the folder `lib` and
317 the distribution of python can vary.
318 
319 Compilation of PyCheMPS2 occurs by linking to the `c++` library in the
320 installation directory. The installation of PyCheMPS2 will fail if that
321 library is not properly installed. If you have pulled a newer version of
322 CheMPS2, please remember to reinstall the `c++` library first, before
323 reinstalling PyCheMPS2!
324 
325 
326 ### 7. Test PyCheMPS2
327 
328 When libchemps2 has been compiled **without MPI**, PyCheMPS2 can be tested
329 by running (remember that the python site-packages folder can vary):
330 
331  > cd /sourcefolder/chemps2/PyCheMPS2/tests
332  > export PYTHONPATH=${PYTHONPATH}:/option3/lib/python2.7/site-packages
333  > python test1.py
334  > python test2.py
335  ...
336 
337 If you compiled the `c++` library with `-DMKL=ON`, you might get the error
338 
339  Intel MKL FATAL ERROR: Cannot load libmkl_avx.so or libmkl_def.so.
340 
341 This issue of using Intel’s MKL inside python is known and reported. To
342 get the python tests to run, you can set the variable `LD_PRELOAD` in
343 order to preload libmkl\_rt.so. On my system, this is done with
344 
345  > export LD_PRELOAD=/opt/intel/mkl/lib/intel64/libmkl_rt.so
346 
347 The python tests do exactly the same thing as the `c++` tests above, and
348 illustrate the usage of the python interface to libchemps2. The tests
349 should end with a line stating whether or not they succeeded. Note that the
350 tests are too small to see (near) linear scaling with the number of cores,
351 although improvement should still be noticeable.
